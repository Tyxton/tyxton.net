Nginx

Switching to nginx from Apache2

Admittedly, I knew little to nothing about the process of hosting a website when I jumped into it, however, I did start with Apache2 when it came to my first server as it was heavily advertised by the Debain communitity. Looking futher into it, It seems like I should've started with nginx for several reasons. SO I'll get started with switching the server over. I think immediately the directory for nginx is different, and I may have to change host files? We'll see.


So as it turns out, that was suprisingly simple, nginx and apache2 configuration files are pretty interchangeable as far as their variables, it's sort of just like... copying it but slightly changing a few things. Here is a sort of translation for the Virtual Host files: 

		
Apache                                 Nginx
------                                 ------

<VirtualHost *:80>                    server {
                                            listen 80;

ServerName yoursite.com
ServerAlias www.yoursite.com           server_name yoursite.com www.yoursite.com;

DocumentRoot /path/to/root             root /path/to/root;

AllowOverride All                      (No Available Alternative)

DirectoryIndex index.php               index index.php;

ErrorLog /path/to/log                  error_log /path/to/log error;

CustomLog /path/to/log combined        access_log /path/to/log main;

Alias /url/ "/path/to/files"           location /url/ {
<Directory "/path/to/files">                 alias /path/to/files;

Most everything works when transfering over, so really you just have to look for the difference in the file names and then adjust accordingly. With this, I am now successfully running everything off of nginx, which is incredibly gratifying. Now to fix everything else that was wrong before.

I think that pretty much everything is good to go now, SSL encryption is the next step.
